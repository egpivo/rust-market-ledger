Overview

This page demonstrates the comparison of different consensus algorithms in a market ledger context.

Consensus Algorithms Compared

  • PBFT - Practical Byzantine Fault Tolerance (BFT, requires majority)
  • Gossip - Epidemic/gossip protocol (no majority voting)
  • Eventual - Time-based commitment (eventual consistency)
  • Quorum-less - Weighted voting based on reputation
  • Flexible Paxos - Flexible quorum Paxos (Q1 intersects with previous Q2)

Key Insights

  • PBFT provides the highest security but sacrifices scalability due to its high latency and low throughput.
  • Gossip excels in decentralization and scalability but has lower security guarantees.
  • Eventual Consistency offers a balance between decentralization and scalability but trades off security.
  • Quorum-less provides a balanced approach across all three dimensions.
  • Flexible Paxos offers strong security with better scalability than PBFT through flexible quorums.
